Leaps is a service for hosting collaboratively edited documents using operational transforms to ensure zero-collision synchronization across any number of editing clients.

No dependencies

Leaps is ready to be dropped onto a server and deployed as a service. It is a stand alone binary with zero dependencies.

Multiple storage options are supported and the list is growing, currently including MySQL, PostgreSQL, file based and in-memory. The optional ability to host static files over HTTP is also included.

Easy to use

Leaps is configured through a single config file, with many simple examples for various use cases.

The client is written in plain old JavaScript and is designed to be easily added into a website, with support for binding to textarea elements, ACE editors and CodeMirror editors with a single line of code.

Quick Tutorial: Pair Programming Over LAN

Leaps has built in capabilities to share your local file system, this can be used to easily and quickly set up pair programming environments over LAN. Here is an example of sharing a source directory tree.

Step one, download and extract

wget https://github.com/Jeffail/leaps/releases/download/v0.1.2/leaps-linux_amd64-v0.1.2.tar.gz -O leaps.tar.gz
tar -xvf ./leaps.tar.gz
cd leaps

Here we have downloaded the linux 64-bit binary, change the url to suit your architecture of choice (find them at github releases).

We are going to install leaps, this copies the binary into /usr/sbin/leaps and some config and static web files into /etc/leaps.

NOTE: Leaps can be installed with the --daemon option which launches it at boot up, we do not want this, so if you have previously installed leaps then uninstall it before proceeding.

sudo ./install.sh

Step two, configure leaps

After installing we ended up with a default config file in /etc/leaps/config.yaml, we need to replace this with a configuration for local file system access. Leaps comes with a premade config file for our purposes in ./config/leaps_share.yaml, so lets copy it:

cp ./config/leaps_share.yaml /etc/leaps/config.yaml

Read the new config and you should see:

logger:
  prefix: '[leaps]'
stats:
  prefix: leaps
  retain_internal: true
curator:
  storage:
    type: file
    store_directory: .
  authenticator:
    type: file
    file_config:
      share_path: .
      path: /files
http_server:
  static_path: /
  socket_path: /socket
  address: :8001
  www_dir: /etc/leaps/www/share_dir

The values of note are curator.storage.store_directory and authenticator.file_config.share_path, these should both point to our desired share path. Currently they both point to the current directory, so by running leaps we will be sharing the directory which we currently reside. You can change these values into any directory or file path, relative or absolute, just make sure they point to the same place.

Another configuration value of interest is http_server.address, this is the HTTP address that leaps should bind to. Currently it binds to port 8001, you might want to change this.

Finally, the value http_server.www_dir points to a path containing our web client static files. Leaps comes packed with a fully functional interface for editing local files in the ACE web editor, it uses a CDN to obtain the ACE js files, if you wish to download a local copy of these files you can do the following:

( cd /etc/leaps/www/share_dir && ./dl_ace.sh )

Step three, choose a directory to share

That's it, you should now be ready to run leaps for sharing files. Navigate to your source code and run leaps:

$ cd ~/src/code/cool_project_to_share
$ leaps

Config file not specified, reading from /etc/leaps/config.yaml
Launching a leaps instance, use CTRL+C to close.

2015-01-20T23:02:31Z [leaps] <INFO> Serving file search API at path: /files
2015-01-20T23:02:31Z [leaps][http] <INFO> Listening for websockets at address: :8001/socket
2015-01-20T23:02:31Z [leaps][http] <INFO> Serving static file requests at address: :8001/
2015-01-20T23:02:31Z [leaps][stats] <INFO> Listening for stats requests at address: localhost:4040/stats

You should see an output something like this, you don't need to specify the configuration file location since leaps checks for the path /etc/leaps/config.yaml.

Now open a browser at the address http://localhost:8001, or whatever you changed the bind address to, and you should see the editing window. You can invite others to edit these files by finding out your LAN IP and giving them the URL http://<yourIPhere>:8001.
